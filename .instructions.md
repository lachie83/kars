# AzureClaw Development Guide

## What is AzureClaw?
A secure AI agent runtime on Azure AKS. Agents run in isolated K8s sandboxes with E2E encrypted inter-agent communication (Signal Protocol via AgentMesh).

## Repository Structure
```
controller/          Rust — K8s controller, reconciles ClawSandbox CRDs
inference-router/    Rust — HTTP/WS sidecar router (AOAI proxy, AGT relay/registry proxy)
cli/                 TypeScript — OpenClaw plugin (tools, AGT SDK integration)
sandbox-images/      Dockerfiles + entrypoints for agent containers
vendor/              Patched upstream components (relay, registry, SDK)
deploy/              K8s manifests, Bicep, Helm charts
policy-engine/       Governance policy profiles
```

## Key Rules

### Images
- Always use `:latest` tag. Do NOT hardcode version tags.
- Controller reads `SANDBOX_IMAGE` / `INFERENCE_ROUTER_IMAGE` env vars but defaults to `:latest`.
- Do NOT set these env vars manually — let the controller use defaults.
- CRD `openclaw.image` field should be empty unless overriding for a specific sandbox.

### Plugin (cli/src/plugin.ts)
- OpenClaw loads the plugin TWICE. Use `process.env.__AGT_INITIALIZED` singleton guard.
- The `onMessage` handler processes incoming E2E encrypted messages and calls the local LLM.
- Trust scoring is in the KNOCK handler — configurable via `AGT_TRUST_THRESHOLD` env var.

### Vendor Patches
- `vendor/` contains patched versions of upstream agentmesh components.
- Changes are documented in each component's README.md.
- Do NOT update from npm/upstream without re-applying patches.
- The SDK dist is overlaid onto `node_modules/@agentmesh/sdk/dist/` in the Docker build.

### Docker Builds
- Sandbox: `docker build --platform linux/amd64 -f sandbox-images/openclaw/Dockerfile -t azureclawacr.azurecr.io/openclaw-sandbox:latest .`
- Relay: `cd vendor/agentmesh-relay && docker build --platform linux/amd64 -t azureclawacr.azurecr.io/agentmesh-relay:latest .`
- Registry: `cd vendor/agentmesh-registry && docker build --platform linux/amd64 -t azureclawacr.azurecr.io/agentmesh-registry:latest .`
- Always push with `az acr login --name azureclawacr` first.

### Testing E2E
1. Ensure agentmesh namespace has relay + registry + postgres pods running
2. Parent sandbox pod must show `AGT mesh connected` in logs
3. Sub-agent needs ~90s after spawn for background relay listener to start
4. Use `azureclaw_mesh_send` to send, `azureclaw_mesh_inbox` to receive
5. Relay logs should show zero message content (only connection events = E2E confirmed)

### Common Issues
- "Cannot find module '@agentmesh/sdk'" → npm install failed in Docker, check vendored SDK overlay
- "SignatureVerificationFailed" in relay → timestamp format mismatch, check vendor/agentmesh-relay patches
- "wrong secret key for the given ciphertext" → ratchet key mismatch, check vendor/agentmesh-sdk session.ts
- Duplicate messages → singleton guard not working, check `__AGT_INITIALIZED` env var
- Sub-agent doesn't receive relay messages → no background agent session, check entrypoint.sh
