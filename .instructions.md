# AzureClaw Development Guide

## What is AzureClaw?
A secure AI agent runtime on Azure AKS. OpenClaw (open-source AI agent framework) runs inside isolated K8s sandbox pods with E2E encrypted inter-agent communication (Signal Protocol via AgentMesh). Each agent gets its own namespace, NetworkPolicy, seccomp profile, and inference router sidecar.

## Current State (March 23, 2026)
- **E2E encrypted relay**: WORKING — full Signal Protocol (X3DH + Double Ratchet)
- **LLM-processed replies**: Sub-agents call their local LLM and reply via encrypted relay
- **Trust scoring**: Wired in KNOCK handler (AGT_TRUST_THRESHOLD env var)
- **Singleton**: Fixed — no duplicate messages from double plugin load
- **Images**: All tagged `:latest` in ACR, controller uses defaults
- **AKS cluster**: `azureclaw-aks` in resource group with 3 node pools (system, clawpool, agentmesh)
- **ACR**: `azureclawacr.azurecr.io`

## Three AgentMesh Repos (Important!)
- `amitayks/agentmesh` — **actual protocol implementation** (relay, registry, SDK) — published as `@agentmesh/sdk` on npm
- `microsoft/agent-governance-toolkit` (AGT) — governance layer (policy, trust, compliance) — depends on `@agentmesh/sdk`
- `agentmesh/agentmesh` (private GitHub org) — canonical source, `amitayks/agentmesh` is the public mirror
- Our `cli/package.json` depends on `@agentmesh/sdk: ^0.1.2` — we overlay vendored dist in Docker

## Key Rules

### Images
- Always use `:latest` tag. Do NOT hardcode version tags (we had v11-v25 tag hell).
- Controller defaults to `:latest` (line 945 of reconciler.rs).
- Do NOT manually `kubectl set env` for SANDBOX_IMAGE/INFERENCE_ROUTER_IMAGE.
- CRD `openclaw.image` should be empty — controller uses default.
- Previous issue: manually set env vars + CRD image fields caused version drift.

### Plugin (cli/src/plugin.ts)
- OpenClaw loads the plugin TWICE (tool registry + agent session).
- Singleton guard: `process.env.__AGT_INITIALIZED = '1'` — only first load creates AGT client.
- `onMessage` handler: receives decrypted messages → calls LLM via sidecar → replies via E2E relay.
- `onKnock` handler: evaluates trust score + policy before accepting sessions.
- `AGT_TRUST_THRESHOLD` env var (default 500, set in entrypoint.sh).
- Auto-reply sends `task_response` with LLM answer back via encrypted relay.

### Sub-Agent Container Lifecycle
- `entrypoint.sh` starts OpenClaw gateway (port 18789) in background.
- Then starts a persistent `openclaw agent --local` session (PID logged, output → /tmp/agt-relay-listener.log).
- This background session loads the plugin → connects to AGT relay → can receive/reply to E2E messages.
- Without this background session, the sub-agent can't receive relay messages (plugin never loads).

### Vendor Patches (8 bugs fixed in upstream @agentmesh/sdk v0.1.2)
All patches documented in vendor/*/README.md:
1. Relay: chrono `to_rfc3339()` Z vs +00:00 → raw string verification
2. Registry: same for `verify_update_signature` → raw string
3. SDK `buildBundle()`: empty signature → async re-sign with identity
4. SDK `buildBundle()`: empty oneTimePrekeys → store + return public keys
5. SDK `base64Decode`/`fromBase64`: crash on `x25519:`/`ed25519:` prefix → strip before atob
6. SDK `initiateSession`: missing peer ratchet key → pass `peerBundle.signedPrekey`
7. SDK `acceptSession`: responder uses fresh keypair → `initializeResponder()` with signedPrekey pair
8. SDK `client.ts`: transport receive not wired + no KNOCK sent + no X3DH params in first message

### Docker Build
- Sandbox Dockerfile installs npm deps, then overlays vendored SDK:
  ```dockerfile
  RUN cd /opt/azureclaw-plugin && npm install --omit=dev --ignore-scripts || true
  RUN mkdir -p /opt/azureclaw-plugin/node_modules/@agentmesh/sdk
  COPY vendor/agentmesh-sdk/dist/ /opt/azureclaw-plugin/node_modules/@agentmesh/sdk/dist/
  COPY vendor/agentmesh-sdk/package.json /opt/azureclaw-plugin/node_modules/@agentmesh/sdk/package.json
  ```
- Also installs vendored SDK's own deps (libsodium-wrappers) via a separate npm install.
- Relay/Registry Dockerfiles need Rust 1.94+ (upstream uses 1.83 which is too old).
- Build from repo root: `docker build -f sandbox-images/openclaw/Dockerfile .` (context = repo root).

### Common Issues & Fixes
| Symptom | Cause | Fix |
|---------|-------|-----|
| "Cannot find module '@agentmesh/sdk'" | npm install failed silently in Docker | Check vendored SDK overlay in Dockerfile |
| "SignatureVerificationFailed" in relay | Timestamp format mismatch | Apply vendor/agentmesh-relay patches |
| "wrong secret key for the given ciphertext" | Ratchet key mismatch | Check vendor/agentmesh-sdk session.ts (initializeResponder) |
| Duplicate messages in inbox | Plugin loaded twice without singleton | Check `__AGT_INITIALIZED` env guard |
| Sub-agent doesn't receive relay messages | No background agent session | Check entrypoint.sh relay listener |
| Old image served despite `:latest` push | AKS node cache | Use `imagePullPolicy: Always` or restart pods |
| Controller doesn't pick up new image | Hardcoded SANDBOX_IMAGE env | Remove env var, let controller use default |
| Prekey upload 401 | Registry timestamp auth bug | Apply vendor/agentmesh-registry patches |
| "Invalid character" in base64 | `x25519:` key prefix | Apply vendor/agentmesh-sdk base64Decode fix |
| Registry SQL error on prekey consume | Ambiguous `prekey_id` column | Known upstream bug, doesn't block E2E |
