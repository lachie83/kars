# Security Audit — Hermes runtime A1.3 + A1.4 (CLI, docker spawn, governance hook)

**Scope**: Three additive changes building on `a7882b8` foundation:
1. CLI `--runtime hermes` recognised everywhere (A1.3a)
2. Docker dev sub-agent spawn becomes runtime-aware (A1.3b)
3. AGT policy gate (`pre_tool_call` hook) implemented in the Hermes plugin (A1.4)

## Changes

### 1. `cli/src/runtime.ts` — `Hermes` added to the runtime registry

- New entry in `RuntimeKind` union (`"Hermes"`) and `RuntimeFlag` union (`"hermes"`).
- `FLAG_TO_KIND` maps `"hermes" → "Hermes"`.
- `WIRED_KINDS` includes `"Hermes"` so `assertRuntimeWired()` accepts it.
- `buildRuntimeBlock()` gains a `"Hermes"` case that emits `{kind: "Hermes", hermes: {}}` for `kars add --runtime hermes`.
- `cli/src/commands/add.ts` — `--runtime` help text updated to include `hermes`.

### 2. `inference-router/src/spawn/docker.rs` — runtime-aware sub-agent spawn

- `docker_create_body()` now reads `KARS_RUNTIME_KIND` from the parent's env (controller sets this in AKS/local-k8s; docker dev parent CLI sets it via `dev.ts` — pending separate commit).
- Image selection switches on the parent runtime kind:
  - `Hermes` → `kars-sandbox-hermes:dev`
  - `PydanticAi` → `kars-sandbox-pydantic-ai:dev`
  - `Anthropic` → `kars-sandbox-anthropic:dev`
  - `LangGraph` → `kars-sandbox-langgraph:dev`
  - default (incl. `OpenClaw`) → `kars-sandbox:dev`
- The `KARS_DEV_IMAGE` env still overrides this when set (back-compat with operators who pin a custom image).
- New envs injected on every spawned child container:
  - `KARS_MODEL=<model>` — generic kars contract name (`OPENCLAW_MODEL` retained for OpenClaw entrypoint compat)
  - `KARS_RUNTIME_CONTRACT_VERSION=v1` — runtime contract assertion (mirrors what the controller injects on the AKS/k8s path)
  - `KARS_RUNTIME_KIND=<kind>` — child's runtime kind (inherited from parent unless `kars_spawn` overrides)
  - `PARENT_RUNTIME_KIND=<kind>` — parent's kind for telemetry/lineage

### 3. `runtimes/hermes/src/kars_runtime_hermes/plugin/governance.py` — pre_tool_call hook

Full implementation replacing the A1.1 stub. ~250 LOC including:

- **Action verb taxonomy** — `_action_verb()` derives the canonical verb per the contract (`shell:`, `egress:`, `memory:`, `mesh:`, `handoff:`, `spawn:create:`, `tool:`).
- **Canonical construction rules** (`_canonicalize()`):
  - Length cap 256 bytes (truncate with `...`)
  - Secret redaction: Bearer tokens, JWT-shaped strings, GitHub PATs (`ghp_*`/`ghs_*`), OpenAI keys (`sk-*`), `api[_-]?key`/`access[_-]?token`/`auth[_-]?token` regex
  - Newline collapse + UTF-8 sanitization
- **`evaluate()`** — calls `POST /agt/evaluate` via the shared `router_client`; returns a `GovernanceDecision` dataclass.
- **Fail-closed grace period** — env-configurable via `KARS_AGT_EVALUATE_FAIL_OPEN_GRACE` (default 3, max 10, 0 = fail-closed immediately). Process-wide counter resets on the first successful response.
- **`_on_pre_tool_call` hook** — Hermes calls this before every tool. Returns `None` on allow (proceed); on deny returns a JSON-string error result that Hermes surfaces to the LLM.
- **`register(ctx)`** wires the hook into Hermes' plugin context.

19 new tests in `runtimes/hermes/tests/test_governance.py`:
- Action verb construction for each verb family (`shell`, `egress`, `memory`, `mesh`, `spawn`, `tool`-default)
- Truncation at 256 bytes
- Secret redaction (bearer, JWT, GitHub, newlines)
- `/agt/evaluate` success path returns `Allow`
- Deny path returns `Deny` with reason + matched_rule
- Grace-period semantics: first N failures pass, then fail-closed
- Counter reset on successful response
- Non-2xx HTTP treated as failure
- Hook returns `None` on allow
- Hook returns JSON-error string on deny
- `register()` wires the hook into the ctx

## Risk Assessment

- **Strictly additive at the CLI / runtime registry layer**: New enum cases, new map entries, new switch arms. No existing runtime's behaviour changes.
- **Docker spawn fallback preserves existing behaviour**: When `KARS_RUNTIME_KIND` is unset (every existing docker dev parent today — until `kars dev` sets it in a follow-up), the image defaults to `kars-sandbox:dev` (OpenClaw), exactly matching pre-change behaviour.
- **Governance hook fails closed under load**: Default grace is 3 failures (matches OpenClaw `FAIL_CLOSED_THRESHOLD`); operators can pin to 0 for immediate fail-closed. Empty action verbs (e.g. tool called with no params) still get evaluated — the action string `tool:<name>:` is well-formed.
- **Secret redaction in action verbs**: Bearer/JWT/GitHub/OpenAI secrets are scrubbed BEFORE they hit the router (defense in depth — the router also redacts at audit time via `agentmesh_mcp::redactor`). Tests pin each redaction pattern.
- **No mesh changes**: This commit doesn't touch the mesh path. `kars_mesh_*` tools still return the Act 2 stub error.
